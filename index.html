Every application starts from the originally loaded target MIDI, so changing a setting never compounds or reuses an older result.

How this actually works (not sidechaining): the source is reduced to a fixed 16-step-per-bar timing/velocity template. Each target note is matched to its nearest step and pulled toward that step's position by Timing Strength — 0% leaves it untouched, 100% puts it exactly on the source's timing for that step, 50% moves it exactly halfway there. How far that actually is in ms depends on how far the note already sits from the grid, then Max Timing Shift (and the tighter Bass Max Shift) hard-caps the result in milliseconds regardless of Strength — so if a note is capped, raising Strength further won't move it any more; you'll need to raise the ms cap instead. Velocity Strength works the same way but on loudness: it nudges each note by the source step's own deviation from its track average, scaled by the slider — it does not copy the source's absolute velocities.
